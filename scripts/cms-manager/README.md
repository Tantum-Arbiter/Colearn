# CMS Manager

Local CLI tool for managing story content before uploading to Firestore and GCS.

## Installation

```bash
cd scripts/cms-manager
npm install
```

## Commands

### List Stories

View all stories with their validation status:

```bash
npm run list
# or
node src/index.js list

# JSON output for scripting
node src/index.js list --json
```

### Validate Stories

Validate stories against the JSON schema:

```bash
# Validate a single story
npm run validate squirrels-snowman

# Validate with verbose error details
npm run validate squirrels-snowman -- --verbose

# Validate all stories
npm run validate -- --all --verbose
```

### Format Stories

Normalize and format story data:

```bash
# Format a single story
npm run format squirrels-snowman

# Preview changes (dry run)
npm run format squirrels-snowman -- --dry-run

# Format all stories
npm run format -- --all
```

### Prepare for Upload

Full pipeline: validate → format → generate asset manifests:

```bash
npm run prepare

# Preview what would happen
npm run prepare -- --dry-run

# Force continue even with validation errors
npm run prepare -- --force
```

### Import from Story Engine

Import a story from the story-engine output:

```bash
# Import from story-engine generated directory
npm run import ../story-engine/generated/my-new-story

# Overwrite existing story
npm run import ../story-engine/generated/my-new-story -- --overwrite
```

## Workflow

### 1. Create/Import Stories

Either create stories manually in `cms-stories/` or import from story-engine:

```bash
npm run import ../../story-engine/generated/new-story
```

### 2. Validate & Format

```bash
npm run validate new-story --verbose
npm run format new-story
```

### 3. Prepare for Upload

```bash
npm run prepare
```

This generates `upload-manifest.json` which lists all stories and assets.

### 4. Commit & Push

```bash
git add scripts/cms-stories/
git commit -m "Add new story: new-story"
git push
```

### 5. GitHub Actions

The CI pipeline will:
1. Upload story metadata to Firestore
2. Upload assets to Google Cloud Storage
3. Update content version for delta-sync

## Validation Rules

The CMS manager validates against `scripts/story-schema.json`:

| Field | Required | Format |
|-------|----------|--------|
| `id` | Yes | kebab-case, e.g. `my-story-name` |
| `title` | Yes | 1-100 characters |
| `category` | Yes | One of: personalized, bedtime, adventure, nature, friendship, learning, fantasy, music, activities, growing |
| `pages` | Yes | Array with at least 1 page |
| `pages[].id` | Yes | kebab-case |
| `pages[].pageNumber` | Yes | Sequential integers starting at 0 |
| `pages[].text` | Yes | Story text |

### Custom Validations

- Sequential page numbering (0, 1, 2, ...)
- Duration matches page count
- Asset paths follow convention
- Localization consistency across pages

## Troubleshooting

**"Story not found"**: Make sure the story directory exists in `scripts/cms-stories/` with a `story-data.json` file.

**Validation errors**: Run with `--verbose` flag to see detailed error messages.

**Asset path warnings**: Ensure image paths start with `assets/stories/{story-id}/`.
